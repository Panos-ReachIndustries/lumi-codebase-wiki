## V2 Modules (Lumi-AI-Core)

- [V2.BackgroundSubtraction](ai-core/modules/BackgroundSubtraction.md) — Counts and groups discrete objects (originally bacterial colonies) inside a circular ROI using adaptive thresholding plus auto-tuned parameters from a VGG16 + regressor.
- [V2.BasicOps](ai-core/modules/BasicOps.md) — Tiny utility module of generic OpenCV helpers shared across V2 modules. Currently just one function — convert a binary mask to a list of polygons.
- [V2.CameraViewAnalyzer](ai-core/modules/CameraViewAnalyzer.md) — Uses GPT-4o vision to pick the best camera for a given subject and detect rotation needs — the bootstrap step for multi-camera monitor setups.
- [V2.CentrifugeAngleAnalyzer](ai-core/modules/CentrifugeAngleAnalyzer.md) — Tracks the rotational angle of a centrifuge rotor through manual rotation and motor spin, and remembers vial positions so each tube can be re-identified after the cycle.
- [V2.CodeReaders](ai-core/modules/CodeReaders.md) — Unified barcode / QR / DataMatrix reader with one V2 wrapper that routes to pyzbar or pylibdmtx backends per code type.
- [V2.Colours](ai-core/modules/Colours.md) — Colour analysis for an image region — dominant colours via K-means, opacity via Gaussian-blur iteration, and homogeneity / contrast via GLCM. Underpins the colour and liquids monitors.
- [V2.Detection](ai-core/modules/Detection.md) — The unified object detection / segmentation API. The most-imported V2 module — read this if you want to understand the V2 contract.
- [V2.Dimensions](ai-core/modules/Dimensions.md) — 3D spatial analysis — lifts 2D bounding boxes into 3D using a depth map, plus real-world distance and area calculations from a known reference. Powers the 3DCalc monitor.
- [V2.DocIngestingPDF](ai-core/modules/DocIngestingPDF.md) — Uploads scientific-protocol PDFs to GPT-4o, then parses the response into a stages → steps → actions → subActions hierarchy used by the protocol builder.
- [V2.Gestures](ai-core/modules/Gestures.md) — Rule-based hand-gesture classifier (okay, thumbs up/down, peace, middle finger) over 21-point hand keypoints from the glove detector.
- [V2.GridTracker](ai-core/modules/GridTracker.md) — Grid-based tracking for laboratory well plates and well racks — detects the grid, tracks individual wells / vials, and exposes per-cell state across frames.
- [V2.HistoricVideoAnalysis](ai-core/modules/HistoricVideoAnalysis.md) — Post-hoc analysis of recorded experiment video — currently houses ActivityDetection, a CV-only "is something happening?" detector that runs without models or APIs.
- [V2.Interactions](ai-core/modules/Interactions.md) — Human-hand interaction primitives — track which hand touches which object over time, and simulate "wiping" coverage of a region.
- [V2.LabContainerTracking](ai-core/modules/LabContainerTracking.md) — Tracks lab containers across frames and maintains an event-sourced substance inventory with provenance queries.
- [V2.Machine](ai-core/modules/Machine.md) — Lazy-loading facade over text recognition, screen detection, and gauge-reading capabilities.
- [V2.ModelInference](ai-core/modules/ModelInference.md) — Standardised model-inference framework that wraps every backbone (YOLO, MiDaS, FCN, MediaPipe pose, etc.) behind a common dict-in/dict-out API.
- [V2.ModularDetector](ai-core/modules/ModularDetector.md) — Task-oriented ONNX bbox detector with built-in ByteTrack tracking — pick a task key, get tracked detections in original-frame pixel space.
- [V2.MultiCamera](ai-core/modules/MultiCamera.md) — Cross-camera instance matcher — links per-camera object IDs to a canonical identity by aligning interaction events across feeds.
- [V2.NuclioRequest](ai-core/modules/NuclioRequest.md) — HTTP client wrapper for Nuclio serverless function endpoints — handles base64 image encoding, mask decoding, and V2-style error responses.
- [V2.ObjectInteractionsManager](ai-core/modules/ObjectInteractionsManager.md) — Routes tracked objects to per-class interaction handlers — the dispatcher that turns a list of tracks into pairwise interaction events.
- [V2.Pipetting](ai-core/modules/Pipetting.md) — Detects Aspirate / Dispense pipetting actions from video using a temporal state machine over hand, tool, and container detections.
- [V2.PoCVIS](ai-core/modules/PoCVIS.md) — Visualisation toolkit for PoC video pipelines — info panels, logo overlays, mask overlays, summary frames, all in OpenCV/PIL.
- [V2.ProtocolBuilder](ai-core/modules/ProtocolBuilder.md) — LLM-backed parser that decomposes scientific protocols into structured stages/steps/actions and generates custom-agent configs. Wraps OpenAI GPT-4o and Google Gemini.
- [V2.ReportGenerator](ai-core/modules/ReportGenerator.md) — Generates LaTeX-rendered PDF reports combining protocol-execution analysis and per-monitor data visualisations.
- [V2.SegmentAnything](ai-core/modules/SegmentAnything.md) — Thin re-export layer for SAM-based vessel segmentation plus a NumPy-to-JSON serialisation helper.
- [V2.TextReader](ai-core/modules/TextReader.md) — Reads text from frames or stills using YOLO digit detection plus an LLM (OpenAI or local Llama) for OCR, with temporal smoothing.
- [V2.ThinLayerChromatography](ai-core/modules/ThinLayerChromatography.md) — Analyses thin-layer chromatography plates from a still image — detects plates, lanes, and spots; returns Rf values and visualisations.
- [V2.Tracking](ai-core/modules/Tracking.md) — Multi-object tracking — class-routed Norfair STM tracker with motion-aware Kalman, appearance re-ID, and a polygon variant. The default downstream of any V2 detector.
- [V2.Utils](ai-core/modules/Utils.md) — Shared image, geometry, and data-conversion helpers used across V2 modules. A leaf dependency — nothing inside V2 should import upward from here.
- [V2.Vessels](ai-core/modules/Vessels.md) — Container detection, segmentation, phase-line detection, height-to-volume conversion, and 3D rendering for lab vessels.
- [V2.VisibleObjectList](ai-core/modules/VisibleObjectList.md) — A tracked-object registry with lifecycle — current visible counts, total ever-seen counts, and double-count prevention. Detection plus an internal IoU tracker plus a counter.
- [V2.Visualiser](ai-core/modules/Visualiser.md) — A Flask-based web debugger that renders pipeline outputs (detections, tracks, poses, polygons, masks) live in a browser. Used heavily by the custom agent.
- [V2.Vortexing](ai-core/modules/Vortexing.md) — Detects active use of a vortexer by tracking sustained glove–vortexer bbox overlap with grace periods and occlusion handling.
