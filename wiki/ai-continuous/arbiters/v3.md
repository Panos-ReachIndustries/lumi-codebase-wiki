# V3 arbiter

V3 is the ground-up redesign of the protocol arbiter, building alongside [v2](v2.md) which keeps running in production until cutover. **Phase 1 ships only the domain layer** ‚Äî pure logic, no I/O, no Kafka, no S3, no orchestrator. If you are looking for the thing that runs in prod, that is [v2](v2.md). V3 is what we are migrating *to*.

## Pure state-machine domain layer

`Lumi-AI-Continuous/protocol_arbiter_v3/domain/` is a hermetically sealed package. The contract is enforced by an import-linter contract registered as a pytest test (`pyproject.toml`):

> `domain/` may import only from the Python standard library. Any third-party package, any other arbiter layer, or any module from the rest of the repo is forbidden.

Concretely, the rules from `protocol_arbiter_v3/domain/README.md`:

1. **Every type is immutable.** `@dataclass(frozen=True)` everywhere. State updates produce new instances via `dataclasses.replace`. See `state.py:16-50`.
2. **Every function is pure.** Same inputs ‚Üí same outputs. No clock reads, no random, no globals. Time is passed in as a `datetime` argument.
3. **Side effects are returned as data.** `apply(state, event) ‚Üí (new_state, list[SideEffect])`. The orchestrator (yet to be written) decides what to do with the side-effect list.

The package is small, deliberate, and complete:

| File | What's inside |
|------|---------------|
| `state.py` | `ProtocolState`, `InstructionState`, `InstructionRuntime` ‚Äî frozen dataclasses, `Mapping`/`tuple` only. |
| `events.py` | The seven `Event` types: `UserStart`, `UserPause`, `UserResume`, `UserMarkComplete`, `UserReset`, `UserAdvance`, `AgentStepComplete`. |
| `side_effects.py` | `LedgerAppend`, `SnapshotWrite`, `UIBroadcast`, `AgentDirectiveUpdate` ‚Äî the things the orchestrator should fire. |
| `transitions.py` | `apply(state, event) ‚Üí (state, [side_effect])`. Total over every (state, event) pair. |
| `instruction_index.py` | Protocol JSON ‚Üí `InstructionIndex`. |
| `world.py`, `world_transitions.py` | `WorldState`, `TrackedObject`, `WorldEvent`; `apply_world(world, event) ‚Üí world`. |
| `predicates.py` | The `Predicate` ADT ‚Äî `And`, `Or`, `Not`, `Sequence`, plus leaves: `ObjectPresent`, `ObjectStableForDuration`, `ObjectMatches`, `InteractionOccurred`, `CodeRead`, `UserMarked`. |
| `instruction_evaluator.py` | Drives the active rule's predicate after every world event; emits `CompletionResult`. |
| `rate_directive.py` | `RateDirective` with per-camera `LaneRate`. |

Compare `protocol_arbiter_v3/domain/state.py:31` with `protocol_arbiter_v2/state.py:41` ‚Äî same idea (`current_instruction_id` plus per-instruction runtime), but every field is frozen and the transition function is total and pure.

## What's missing

V3 has not yet been wired into anything. From `protocol_arbiter_v3/README.md`:

- `effects/` ‚Äî Kafka / S3 / ai-core / UI gateway adapters behind protocols. *(Stubs exist under `protocol_arbiter_v3/effects/` but no integration.)*
- `lifecycle/` ‚Äî boot, recovery, graceful shutdown.
- `fleet/` ‚Äî agent fleet planner, supervisor, rate controller.
- `analysis/` ‚Äî Gemini protocol analyser that emits `Predicate` trees.
- `frontend_object_handlers/` ‚Äî well-plate-style approval flows.
- `arbiter.py`, `main.py` ‚Äî orchestrator and CLI entrypoint.

There is **no equivalent of v2's run loop yet**. You cannot run v3 against Kafka. You can only call `apply(state, event)` from a test.

## Where it fits in the v1 ‚Üí v2 ‚Üí v3 evolution

- [v1](v1.md) ‚Äî nested stage/step/action; Kafka topics + S3 + control flow all interleaved in one 1500-line `arbiter.py`.
- [v2](v2.md) ‚Äî flat instruction IDs; standalone v2-local helpers; same external wire format. Production today.
- **v3** ‚Äî strict layering (`domain/` ‚Üî `effects/` ‚Üî orchestrator), immutable state, predicate ADT replaces ad-hoc block evaluation, side effects as data. Only `domain/` is built.

The full design lives in `Lumi-AI-Continuous/MULTI_AGENT_REDESIGN.md` ‚Äî read that before adding code.

## How to read the domain code

Start at `domain/transitions.py:37`: the `apply` dispatcher is the single source of truth for valid transitions. From there, follow each `_user_*` / `_agent_step_complete` helper. Then read `domain/predicates.py` to see how completion is *expressed* (not evaluated against I/O), and `domain/instruction_evaluator.py` to see the only mutable piece in the layer ‚Äî the latched current rule.

## Tests

`protocol_arbiter_v3/tests/unit/` is the Phase 1 verification suite ‚Äî runs in under 5 seconds via `make check`:

- `test_state.py`, `test_transitions.py`, `test_invariants.py`, `test_replay.py`, `test_edge_cases.py` ‚Äî the state-machine matrix.
- `test_predicates.py`, `test_instruction_evaluator.py`, `test_instruction_index.py` ‚Äî predicate semantics.
- `test_world.py`, `test_world_transitions.py` ‚Äî world model.
- `test_rate_directive.py` ‚Äî fleet-rate directives.
- `test_layer_purity.py` ‚Äî the import-linter contract that fails CI on any leak from `domain/` to a non-stdlib module.

`protocol_arbiter_v3/tests/component/` covers the in-progress `effects/` adapters (in-memory message bus, fake S3 ledger, etc.) but nothing is wired into `domain/`.
